# C++ Money Transfer Web-Service

Высокопроизводительный микросервис для управления банковскими счетами и проведения транзакций. Проект написан на **современном C++17** с акцентом на чистую архитектуру, ручное управление ресурсами (RAII) и промышленную надежность.

## 🛠 Технологический стек

*   **Язык**: C++17 (стандарт индустрии).
*   **Web-Framework**: [Crow](https://crowcpp.org/) — микрофреймворк, вдохновленный Flask.
*   **Networking**: [Asio](https://think-async.com/Asio/) (Standalone) — библиотека для асинхронного ввода-вывода.
*   **Database**: PostgreSQL (в Docker-контейнере).
*   **Драйвер БД**: Нативный C-клиент `libpq`.
*   **Сборка**: CMake 3.10+.
*   **Конфигурация**: JSON ([nlohmann/json](https://github.com/nlohmann/json), интегрированный в Crow).

## 🚀 Основные возможности

*   **CRUD счетов**: Создание нового аккаунта, поиск информации по ID.
*   **Депозиты**: Пополнение баланса с валидацией входных данных.
*   **Транзакции**: Безопасный перевод средств между счетами с использованием SQL-транзакций (`BEGIN`/`COMMIT`/`ROLLBACK`).
*   **Автоматические миграции**: Встроенный `MigrationRunner` автоматически применяет SQL-скрипты при старте приложения, отслеживая состояние схемы в таблице `database_changelog`.
*   **Fail-Fast конфигурация**: Приложение проверяет наличие и валидность всех необходимых настроек (БД, порт) сразу при запуске.

## 📂 Структура проекта

*   `include/` — заголовочные файлы (`.h`).
*   `src/service/` — бизнес-логика (реализация PostgreSQL через `libpq`).
*   `src/controller/` — обработка HTTP-запросов и маршрутизация.
*   `src/configuration/` — логика инициализации приложения, парсинг конфигов и мигратор.
*   `resources/` — статические ресурсы (`application.json`, SQL-миграции).
*   `external/` — внешние зависимости.

## ⚙️ Установка и запуск

### 1. Подготовка окружения
Убедитесь, что у вас установлены:
*   Компилятор с поддержкой C++17 (GCC/MinGW, Clang или MSVC).
*   CMake.
*   Docker & Docker Compose.

### 2. Запуск базы данных
```bash
docker-compose up -d
```

### 3. Настройка приложения
Отредактируйте `application.json` в корне проекта или в папке `resources/`:
```json
{
  "database": { "url": "postgresql://user:pass@host:port/dbname" },
  "server": { "port": 18080 },
  "migrations": { "dir": "resources/migrations" }
}
```

### 4. Сборка и запуск
```bash
mkdir build && cd build
cmake ..
cmake --build .
./web_service
```
*Примечание: При запуске из IDE убедитесь, что **Working Directory** установлена в корень проекта.*

## 🧩 Архитектурные особенности

### RAII и управление ресурсами (DbUtil)
В проекте реализованы собственные RAII-обертки для объектов `libpq`, исключающие утечки памяти:
*   `DbConnection` — автоматически закрывает соединение с БД.
*   `ResPtr` — гарантирует вызов `PQclear` для результатов SQL-запросов.

### Configuration Data Mapping
Применяется паттерн **Data Mapping**: JSON-конфигурация парсится один раз при старте и перекладывается в строго типизированные C++ структуры. Это обеспечивает проверку типов на этапе компиляции и мгновенный доступ к настройкам без парсинга строковых ключей.

## 🌐 API Эндпоинты

### Создание аккаунта: **POST** `/account` (`{ "name": "Ivan" }`)
### Пополнение: **POST** `/account/1/deposit` (`{ "amount": 100.0 }`)
### Перевод: **POST** `/account/transfer` (`{ "from": 1, "to": 2, "amount": 50 }`)
### Инфо: **GET** `/account/1`
