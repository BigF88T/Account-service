# C++ Money Transfer Web-Service

Высокопроизводительный микросервис для управления банковскими счетами и проведения транзакций. Проект написан на **современном C++17** с акцентом на ручное управление ресурсами (RAII) и высокую скорость работы.

## 🛠 Технологический стек

*   **Язык**: C++17 (стандарт индустрии).
*   **Web-Framework**: [Crow](https://crowcpp.org/) — микрофреймворк, вдохновленный Flask.
*   **Networking**: [Asio](https://think-async.com/Asio/) (Standalone) — библиотека для асинхронного ввода-вывода.
*   **Database**: PostgreSQL (в Docker-контейнере).
*   **Драйвер БД**: Нативный C-клиент `libpq`.
*   **Сборка**: CMake 3.10+.
*   **Формат данных**: JSON ([nlohmann/json](https://github.com/nlohmann/json)).

## 🚀 Основные возможности

*   **CRUD счетов**: Создание нового аккаунта, поиск информации по ID.
*   **Депозиты**: Пополнение баланса с валидацией входных данных.
*   **Транзакции**: Безопасный перевод средств между счетами с использованием SQL-транзакций (`BEGIN`/`COMMIT`/`ROLLBACK`).
*   **Безопасность**: Защита от SQL-инъекций через параметризованные запросы (`PQexecParams`).
*   **RAII**: Собственные обертки для автоматического управления временем жизни соединений и результатов запросов (никаких утечек памяти).

## 📂 Структура проекта

*   `include/` — заголовочные файлы (`.h`), описание интерфейсов и DTO.
*   `src/service/` — бизнес-логика (реализации In-Memory и PostgreSQL).
*   `src/controller/` — обработка HTTP-запросов и маршрутизация.
*   `external/` — внешние зависимости (Asio, Crow, PostgreSQL SDK).

## ⚙️ Установка и запуск

### 1. Подготовка окружения
Убедитесь, что у вас установлены:
*   Компилятор с поддержкой C++17 (GCC/MinGW, Clang или MSVC).
*   CMake.
*   Docker & Docker Compose.

### 2. Запуск базы данных
```bash
docker-compose up -d
```
*База будет доступна на порту `5444`.*

### 3. Подготовка зависимостей
Для сборки проекта необходимо наличие библиотек в папке `external/`:
*   `external/Crow` и `external/asio` (header-only).
*   `external/pgsql` (заголовочные файлы и бинарники PostgreSQL для вашей ОС).

### 4. Сборка и запуск
```bash
mkdir build && cd build
cmake ..
cmake --build .
./web_service
```

**GET** `/account/1`

## 🧩 Архитектурные особенности (RAII)

В проекте реализован паттерн **RAII** для управления ресурсами PostgreSQL. Вместо ручного вызова `PQfinish` и `PQclear`, используются специальные структуры-обертки:

```cpp
// Пример использования в коде
{
    DbConnection db(conn_str); // Соединение откроется здесь
    PGresult *res = PQexec(db.conn, "SELECT...");
    ResPtr guard(res); // Память очистится автоматически при выходе из {}
} 
```
Это гарантирует отсутствие утечек памяти даже при возникновении исключений или раннем выходе из функций.
